1. To check if a decimal number `n` has consecutive zeros in its binary representation, we can use the following formula:

   (n^mask) & ((n^mask) >> 1) == 0

   n: The decimal number we want to check.
   mask: A bitmask of length same as the binary representation of n, that has 1s in every bit position

   If the result of the above expression is 0, it means that there are no consecutive zeros in the binary representation of n. Otherwise, if the result is not 0, it indicates that there are consecutive zeros present.

2. To check the rightmost set bit (1) in a binary number `n`, we can use the following formula:

   n & (-n)

   n: The decimal number we want to check.
   -n: The two's complement of n, which can be calculated as ~n + 1.

   This expression will return a number that has only the rightmost set bit of n. If n is 0, it will return 0. Otherwise, it will return a number that represents the rightmost set bit in the binary representation of n.

3. To check the rightmost unset bit (0) in a binary number `n`, we can use the following formula:

   ~n & -(~n)

   n: The decimal number we want to check.
   ~n: The bitwise NOT of n, which flips all the bits of n.
   -(~n): This computes the two's complement of ~n.


   This expression will return a number that has only one bit set, which corresponds to the rightmost unset bit of n. If n is -1 (all bits are set), it will return 0. Otherwise, it will return a number that represents the rightmost unset bit in the binary representation of n.

4. To toggle the bits of a binary number `n`, we can use the following formula:

   n ^ mask

   n: The decimal number we want to toggle.
   mask: A bitmask of same number of bits as n, and that has 1s in all the bit positions of n.

   This expression will return a new number where the bits in n that correspond to the 1s in the mask are toggled (i.e., changed from 0 to 1 or from 1 to 0). The bits in n that correspond to the 0s in the mask will remain unchanged.
